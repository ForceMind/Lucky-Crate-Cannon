# Lucky Crate Cannon

**Lucky Crate Cannon** 是一个现代化的 H5 游戏前后端完整解决方案，从原本的纯前端 Canvas Demo 升级而来，现在配备了 Node.js 后端服务、智能动态风控系统、多语言支持以及一键防冲突自动化部署脚本。

## 🎯 核心功能特性

### 1. 游戏交互
- **现代化 UI**：技能操作采用屏幕右下角圆形图标，更符合手游习惯。
- **智能自动锁定**：当锁定的箱子被击破时，系统将自动扫描屏幕并在存活目标中**优先锁定当前倍率（multi）最高**的箱子，彻底解放双手。
- **国际化双语支持 (i18n)**：游戏默认以全英文显示，屏幕右上角提供无缝的 `EN / 中` 一键切换按钮。

### 2. 强力防作弊后端 (`server.js`)
前端已完全剥离概率计算逻辑，子弹击中目标后的“是否击破”全权交由后端接口 `/api/hit` 判断，彻底杜绝前端修改代码作弊。

### 3. 智能风控与概率调节控制台 (`admin.html`)
自带可视化的后台管理页面（访问 `http://你的IP:端口/admin`）：
- **实时数据看板**：每 2 秒自动刷新“总发射次数”、“总击破数”、“消耗与产出金币”、“玩家实际综合爆率”以及“返奖率 (RTP)”。
- **手动百分比调控**：可以直观地设定“平台概率优势”，例如输入 `3` 表示强行砍掉玩家 3% 的胜率，输入 `-2` 表示平台让利 2%。
- **自动动态风控 (Auto-RTP Control)**：
  - **防大出血拦截**：如果检测到玩家当下周期的实际 RTP 高于 `120%`，平台优势瞬间自动提升至 `20%` 压制玩家。
  - **防流失放水**：系统每 30 秒进行一次周期结算，如果发现过去 30 秒内玩家运气极差（RTP 低于 `80%`），系统会自动将平台优势调低至 `-15%`，给玩家一波甜头。

## 🚀 部署指南 (Linux 一键部署)

我们提供了一个超级智能的脚本 `deploy.sh`，可实现全自动安全部署。

### 脚本特性
- **自动防冲突找端口**：如果默认的 `3000` 端口被占用，脚本会自动递增扫描寻找空闲端口，绝对不会因为端口被抢占而部署失败！
- **自动关闭旧进程**：更新时如果发现老版本进程仍在运行，会自动优雅关闭老进程再部署。
- **自动拉取更新**：如果检测到是 Git 仓库，会自动执行 `git pull` 同步最新代码。
- **自动代码混淆**：自带 `javascript-obfuscator`，在服务启动前会将核心前端逻辑 `game.js` 混淆加密，防止破解。
- **全系 Linux 发行版适配**：自动识别 `apt`, `yum`, `dnf`, `pacman`, `zypper` 并安装依赖库与 Node.js 18 环境。
- **Systemd 守护**：生成原生 `lucky-crate.service` 进程守护文件，保障开机自启。

### 部署步骤
1. 将项目代码传至你的 Linux 服务器。
2. 在项目根目录下执行：
   ```bash
   chmod +x deploy.sh
   sudo ./deploy.sh
   ```
3. 等待脚本执行完毕，终端会打印出**成功分配的端口号**以及**前后台访问链接**。

### 服务管理命令
部署完成后，你可以使用原生 Systemd 命令管理服务：
- **查看运行状态**：`sudo systemctl status lucky-crate`
- **查看实时日志**：`sudo journalctl -u lucky-crate -f`
- **重启游戏服务**：`sudo systemctl restart lucky-crate`
